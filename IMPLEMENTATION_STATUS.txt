================================================================================
SCHEMA OUTPUT (4-PART BUNDLE)
================================================================================

The function generates exactly 4 schema.org objects:

1. SERVICE SCHEMA (Type: Service)
   ├── name: "[Service] in [Location]" or "[Service] in [Subarea], [Location]"
   ├── areaServed: All zip codes (or city if subarea provided)
   ├── potentialAction: ReserveAction linking to contact
   └── provider: MedicalBusiness details

2. LOCAL BUSINESS SCHEMA (Types: LocalBusiness, MedicalBusiness)
   ├── geo: GeoCoordinates with latitude/longitude
   ├── address: Full postal address with location
   ├── areaServed: All service area zip codes
   ├── openingHoursSpecification: 24/7 availability
   └── availableService: Linked service details

3. BREADCRUMB LIST SCHEMA (Type: BreadcrumbList)
   Without subarea (4 items):
   ├── 1. Home
   ├── 2. Locations
   ├── 3. [Location]
   └── 4. [Service] in [Location]

   With subarea (5 items):
   ├── 1. Home
   ├── 2. Locations
   ├── 3. [Location]
   ├── 4. [Subarea]
   └── 5. [Service] in [Subarea], [Location]

4. FAQ SCHEMA (Type: FAQPage)
   Auto-generates 5 location-specific questions:
   ├── 1. What [service] services are available in [location]?
   ├── 2. How much does [service] cost in [location]?
   ├── 3. Do you provide 24/7 [service] in [location]?
   ├── 4. Are caregivers in [location] licensed and trained?
   └── 5. Is [service] available in my area of [location]?

  import { services, locations } from "@/lib/content-data";

  const service = services.find(s => s.slug === "skilled-nursing");
  const location = locations.find(l => l.slug === "san-diego");

  const schemas = generateLocationServiceSchema(

      coordinates: location.coordinates
